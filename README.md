- 概要
	- マリオの生みの親である宮本 茂氏曰く、簡単にできる操作を「２つ同時にやろう」とすると意外と難しく、失敗すると悔しい。そんなところにゲームの面白さがあるとか、なんとか。ほんとかな？試してみよう。 テトリスとブロック崩しで。
	- ”実行ファイル”ディレクトリにある”DXlib_CS.exe”を実行することでゲーム開始
	- Trainingモードではユーザの緊張度を測り最適な難易度に調整するAIを搭載し、ユーザにとって遊びやすい難易度でゲームを遊ぶことができる。
- 必要動作環境
	- Windows98以降のOS
	- DirectX9以降のインストール
	- .NET Framework 4.0以降のインストール
	- ゲームの実行ファイルとDxLib.dllを同じフォルダに入れること。

- 成果物に対しての自身の評価
	- 当初の目的であった、ゲームの面白さの確認はできたと考える。
		- 今回のゲームを通して私が感じたゲームの面白さというのは、「ユーザが自分の”失敗理由を認識”し、次のプレイでは”失敗しないよう改善”することで”自分の成長を実感”できるから、結果的に”面白く感じる”」ということではないかと考える。
	- 改善点としては以下の点があげられる。
		- サウンドの追加
		- テトリスにおける落下してくるブロックを乱数で選択するのではなく、ブロックの出現回数から選択することで出現頻度が偏らないよう修正を行う。
	- また、Trainingモードではユーザの緊張度を計算することで難易度の調整を行い、プレイヤーがより長時間遊べるようにすることでプレイヤーが成長できるよう調整した。
		- このとき緊張度は、テトリスのミノを回転させる行為やブロック崩しのボールをぎりぎりはじき返すなどの行為によって計測する。
		- ここで計測している緊張度におけるパラメータや増加率にはまだまだ改善の余地があり、いろんなユーザに遊んでもらいフィードバックを得る必要がある。
